В файле RegressionSetup.m содержится набор сценариев (сетап-параметров) для тестирования.
В этот же файл необходимо добавлять новые сценарии с соответствующим порядковым номером, при этом кратко комментируя, что данный сценарий проверяет.
Важно: добавление новых сценариев для тестирования в сетап файл RegressionSetup.m осуществляется последовательно в конец файла.

В директории \ReferenceResults содержатся .mat файлы с опорными результатами работы модели, с которыми будут сравниваться результаты запуска тестов.
Результаты запуска тестов сохраняются в директорию \TestResults.

Запуск тестирования осуществляется путём запуска скрипта RegressionTest.m, по окончании расчётов в командное окно будет выведено сообщение о том, какие тесты выполнились успешно.
В случае, когда все тесты успешно проходят, появляется информационное сообщение с предложением обновить опорные результаты.
Если согласиться (нажать «y / yes»), то результаты тестов станут опорными результатами, а папка со старыми опорными результатами будет удалена.
Это необходимо для того, чтобы можно было автоматически добавлять результаты тестирования новых сценариев в папку с опорными данными.

В случае, когда не все тесты будут успешно пройдены, автоматическое обновление папки с опорными результатами не будет произведено.
Если были добавлены новые сценарий в файл RegressionSetup.m, которых нет в папке с опорными результатами, то в командном окне появится предупреждение об этом.
Важно: если вами не были добавлены новые сценарии для тестов в сетап файл RegressionSetup.m, а предупреждение всё равно появилось,
то скорее всего папка с опорными результатами была некорректно изменена, либо могла быть обновлена, необходима дополнительная проверка папки с опорными данными.

Если появилось сообщение с ошибкой «Количество результатов тестирования меньше количества опорных результатов!», то скорее всего сетап файл для тестирования был некорректно изменён, необходима проверка файла RegressionSetup.m

Если в ходе запуска скрипта произошла ошибка в работе модели и/или остановка работы скрипта, то файл RegressionSetup.m будет иметь название SetupRegression.m,
а все остальные файлы сетапы, лежащие в директории со скриптом, будут иметь названия вида _Setup…m.
Это нормальная ситуация, и в скрипте для тестирования учтены подобные вылеты модели.
Достаточно снова запустить скрипт-тестировщик (после устранения обнаруженных проблем в самой модели) и дождаться успешного завершения его работы.

Если необходимо вернуть состояние рабочей директории в исходное (в частности, восстановить все существующие сетап файлы, не относящиеся к тестированию, т.е. убрать символ «_» из их названия),
при этом, допустим, нет необходимости запускать и ждать результаты тестов, то можно запустить секцию в скрипте-тестировщике с помощью Run Section,
название секции «Деактивация файла сетапа для тестов и активация выключенных сетапов».

Также, отдельно можно запустить секцию «Исследование результатов тестов».

Обратите внимание, что скрипт в ходе работы изменяет название файла-сетапа для тестов.
В связи с этим, не рекомендуется вносить изменения в файл-сетап во время работы скрипта, т.к. они просто могут не сохраниться.
